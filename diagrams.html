1. Authentication (Login)

sequenceDiagram
    actor User
    participant Frontend as React Frontend
    participant API as FastAPI Backend
    participant DB as PostgreSQL
    participant JWT as JWT Module

    User->>Frontend: Enter email + password
    Frontend->>API: POST /api/login
    API->>DB: SELECT user WHERE email = ?
    DB-->>API: User row

    alt User not found
        API-->>Frontend: 401 Invalid email or password
        Frontend-->>User: Show error message
    else Password incorrect
        API->>API: bcrypt.checkpw() FAIL
        API-->>Frontend: 401 Invalid email or password
        Frontend-->>User: Show error message
    else Success
        API->>API: bcrypt.checkpw() OK
        API->>JWT: create_access_token(email, user_id)
        JWT-->>API: JWT Token (2h expiry)
        API-->>Frontend: 200 access_token
        Frontend->>Frontend: localStorage.setItem(token)
        Frontend-->>User: Redirect to Dashboard
    end

2. Update Profile

sequenceDiagram
    actor User
    participant Frontend as React Frontend
    participant API as FastAPI Backend
    participant DB as PostgreSQL

    User->>Frontend: Click AD avatar icon
    Frontend->>Frontend: setActiveTab(profile)
    Frontend->>API: GET /api/profile (Bearer token)
    API->>API: Decode JWT - extract email
    API->>DB: SELECT user WHERE email = ?
    DB-->>API: User (name, email, alert_email)
    API-->>Frontend: 200 Profile data
    Frontend-->>User: Display Profile page

    User->>Frontend: Edit name / alert_email
    User->>Frontend: Click Save Changes
    Frontend->>API: PUT /api/profile (Bearer token)
    API->>API: Decode JWT - extract email
    API->>DB: UPDATE users SET name, alert_email
    DB-->>API: Updated user row
    API-->>Frontend: 200 Updated profile
    Frontend-->>User: Profile updated successfully

4. Data Processing (Log Ingestion + ML Anomaly Detection)

sequenceDiagram
    actor Agent as OS Agent
    participant WinLog as Windows Event Logs
    participant API as FastAPI Backend
    participant DB as PostgreSQL
    participant ML as ML Service - Isolation Forest
    participant Notif as Notification Engine
    participant Email as SMTP Server

    loop Every 5 seconds
        Agent->>WinLog: Read Application/System/Security logs
        WinLog-->>Agent: New event entries
        Agent->>Agent: Pre-filter benign noise
        Agent->>API: POST /api/logs/ingest (X-API-Key)

        API->>API: Verify API Key - resolve project_id
        API->>DB: INSERT INTO logs (batch)
        DB-->>API: New log entries with IDs

        loop For each log (BackgroundTask)
            API->>ML: process_log_anomaly(log_id, features)
            ML->>ML: Build 25-feature vector
            ML->>ML: scaler.transform(features)
            ML->>ML: model.decision_function(vector)

            alt Score below -0.0900 - ANOMALY
                ML->>DB: UPDATE logs SET is_anomaly=true
                ML->>DB: SELECT alert_email FROM users
                DB-->>ML: alert_email
                ML->>Notif: send_alert(type, project, details, email)

                alt Cooldown expired or new category
                    Notif->>Email: Send HTML email alert
                    Email-->>Notif: Email delivered
                else Same category within 5 min
                    Notif->>Notif: Throttled - skip
                end

            else Score above threshold - NORMAL
                ML->>DB: DELETE FROM logs WHERE id=?
            end
        end

        API-->>Agent: 200 success
    end
